     INPUT: (env_regs), (prot_areas), (site_coords)
     
     DEPENDENCIES:

     OUTPUT:
     ├─ NO: (NOT_FEASIBLE) (Constraint)
     └─ YES: Continue to 2

2. [LAND_AREA] Is sufficient, accessible land area available for facilities
     and buffer zones?
     
     INPUT: (max_available_area)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ NO: (INJECTION_WELLS) (Small area -remove surfae recharge methods)
     │  → Continue to 2.1
     └─ YES: (SURFACE_RECHARGE) (Adequate area for surface recharge)
        → Continue to 2.1

----------------------------------------------------------------------
SECTION 2: AQUIFER CHARACTERISTICS
----------------------------------------------------------------------

2.1 [AQ_TYPE] What is the aquifer type?
     
     INPUT: (geo_logs), (aq_char), (hydro_reports)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ A. UNCONFINED/PHREATIC AQUIFER → Continue to 2.1.1
     ├─ B. CONFINED/ARTESIAN AQUIFER → Continue to 2.2
     └─ C. UNSUITABLE (e.g., Karstic/Fractured)
         → (NOT_FEASIBLE) (Geological Limitation)

2.1.1 [GW_DEPTH] (Unconfined) What is the depth to water table?
         
         INPUT: (gw_level), (well_logs), (piezo_data)
         
         DEPENDENCIES:
         
         LOGIC:
         
         OUTPUT:
         ├─ A. (SHALLOW) (depth < 20m) → Continue to 2.1.2
         └─ B. (DEEP) (depth >= 20m) → Continue to 2.1.3

2.1.2 [K_SHALLOW] (Shallow) What is the hydraulic conductivity (K)?
         
         INPUT: (pump_test), (slug_test), (lab_perm)
         
         DEPENDENCIES:
         
         LOGIC:
         
         OUTPUT:
         ├─ K > 33 ft/day (High permeability)
         │  → (HIGHLY_FEASIBLE) → Continue to 2.4
         ├─ K = 3.3 - 33 ft/day (Moderate permeability)
         │  → (FEASIBLE_WITH_CONDITIONS) (May require careful design)
         │  → Continue to 2.4
         └─ K < 3.3 ft/day (Low permeability)
            → (NOT_FEASIBLE) (Poor recharge capacity)

2.1.3 [LIMIT_LAYER] (Deep) FEASIBILITY CHECK: Limiting Infiltration Layer?
     
     INPUT: (K_table), (gw_level), (vadose_strat)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     Is the minimum K in the vadose zone < 0.3 ft/day?
         
         ├─ YES - Limiting Infiltration Layer Present
         │  → Continue to 2.1.3.1 (Mitigation Check)
         │
         └─ NO - Permeable Vadose Zone (K_min >= 0.3 ft/day)
             → (FEASIBLE_WITH_CONDITIONS) (Surface or deep methods
             potentially viable) → Continue to 2.1.5 (Infiltration Rate Calculation)

2.1.3.1 [MITIGATION] (Mitigation Check) Is the limiting layer the TOP layer AND < 3 ft thick?
         
         INPUT: (layer_thick), (excav_cost), (layer_pos)
         
         DEPENDENCIES:
         
         LOGIC:
         
         OUTPUT:
         ├─ YES: (FEASIBLE_WITH_CONDITIONS) (Earthwork/Removal Viable)
         │  (Surface spreading viable after low-K layer removal)
         │  → Continue to 2.1.5 (Infiltration Rate Calculation)
         └─ NO: (NOT_FEASIBLE) (Surface Spreading NOT FEASIBLE)
            (Low-K layer is too thick or too deep for economic removal)
            (Deep injection likely required)
            → Continue to 2.1.4 (Feasibility Follow-up)

2.1.4 [DEEP_ACCESS] (Feasibility Follow-up) Can deep injection access the target aquifer?
         
         INPUT: (drill_feas), (aq_depth), (well_cost)
         
         DEPENDENCIES:
         
         LOGIC:
         
         OUTPUT:
         ├─ NO: (NOT_FEASIBLE) (No viable recharge method)
         └─ YES: (FEASIBLE_WITH_CONDITIONS) (Injection required) → Continue to 2.4

2.1.5 [INFIL_RATE] (Infiltration Rate Calculation) Calculate maximum sustainable infiltration rate
     
     INPUT: (K_min), (recharge_dim), (infil_design), (safety_factor)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     Is the calculated infiltration rate adequate for the intended recharge volume?
         ├─ YES: Continue to 2.1.6 (Mound Height Estimation)
         └─ NO: (FEASIBILITY_DOUBTFUL) (Low infiltration capacity)
             → Continue to 2.1.6 (Mound Height Estimation)

2.1.6 [MOUND_HEIGHT] (Mound Height Estimation) Estimate groundwater mound height using Hantush 1967 method
     
     INPUT: (gw_avg), (gw_gradient), (infil_rate), (aq_params), (recharge_geom)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     Is the estimated mound height acceptable for the site conditions?
         ├─ YES: Continue to 2.1.7 (Determine D_min_GW)
         └─ NO: (FEASIBILITY_DOUBTFUL) (Excessive mounding risk)
             → Continue to 2.1.7 (Determine D_min_GW)

2.1.7 [D_MIN_GW] DETERMINE MINIMUM ALLOWABLE DEPTH TO GW (D_min_GW)
     
     INPUT: (D_min_GW), (geotech_reports), (landuse_req), (soil_type), (foundation_depth)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     Is D_min_GW provided?
         ├─ NO: (FEASIBLE_WITH_CONDITIONS)
         │  (Default D_min_GW = 5 ft used)
         │  → ADD RECOMMENDATION (R1): Perform detailed geotechnical/soil survey 
         │     to determine site-specific D_min_GW.
         │  → Continue to 2.1.8 (Potential Storage Estimation)
         └─ YES: Continue to 2.1.8 (Potential Storage Estimation)

2.1.8 [STORAGE_CALC] (Potential Storage Estimation) Is the Vadose Zone Storage less than the minimum
     user-specified storage?
     
     INPUT: (D_min_GW), (layer_thick), (Sy), (gw_current), (gw_seasonal), (storage_min)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ YES: (FEASIBILITY_DOUBTFUL) (Low Storage Capacity)
     │  → Continue to 2.4
     └─ NO: (FEASIBLE) (Adequate storage volume) → Continue to 2.4

2.2 [CONF_ACCESS] (Confined) Is direct aquifer access possible?
     
     INPUT: (drill_feas), (aq_depth), (well_cost), (deep_well_regs)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ NO: (NOT_FEASIBLE) (Cannot access aquifer economically)
     └─ YES: Continue to 2.3

2.3 [PIEZO_HEAD] (Confined) Check piezometric head.
     
     INPUT: (piezo_head), (gs_elev), (aq_pressure)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ Head below ground surface
     │  → (FEASIBLE) (Suitable pressure conditions) → Continue to 2.4
     └─ Artesian conditions (head above surface)
         → (FEASIBLE_WITH_CONDITIONS) (Requires pressure management)
         → Continue to 2.4

2.4 [STORAGE_ADEQ] Is the aquifer storage capacity adequate to
     justify the MAR investment?
     
     INPUT: (storage_vol), (mar_invest), (storage_vol), (econ_analysis)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ A. Adequate storage volume → Continue to 2.5
     └─ B. Limited storage capacity → (NOT_FEASIBLE) (Insufficient volume)

2.5 [HYDRO_RISKS] Are there unacceptable hydrogeological risks?
     (e.g., salt water intrusion, capture by low-priority receptors)
     
     INPUT: (risk_data), (swi_studies), (receptor_analysis), (hydro_model)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ YES: (FEASIBILITY_DOUBTFUL) (Hydrogeological Conflict)
     └─ NO: Continue to 3.1

----------------------------------------------------------------------
SECTION 3: WATER AVAILABILITY AND QUALITY
----------------------------------------------------------------------

3.1 [WATER_AVAIL] Is a reliable source of water available in sufficient quantity for
     long-term MAR operations?
     
     INPUT: (water_avail), (flow_meas), (water_proj), (op_req)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ NO: (NOT_FEASIBLE) (Operational Constraint)
     └─ YES: Continue to 3.2

3.2 [SOURCE_QUALITY] Does the source water quality meet recharge standards without expensive
     treatment (low suspended solids)?
     
     INPUT: (source_quality), (recharge_std), (treatment_cost), (suspended_solids)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ YES: Continue to 3.3
     └─ NO: (FEASIBILITY_DOUBTFUL) (High Treatment Cost)
         ├─ Follow-up: Is pre-treatment economically viable?
         │  ├─ NO: (NOT_FEASIBLE) (Economic/Quality Constraint)
         │  └─ YES: (FEASIBLE_WITH_CONDITIONS) → Continue to 3.3

3.3 [NATIVE_QUALITY] What is the native groundwater quality?
     
     INPUT: (native_quality), (quality_std), (mixing_analysis), (isolation_feas)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ A. Good quality (potable standards) → Continue to 3.4
     └─ B. Poor quality (saline/contaminated)
         ├─ Follow-up: Can isolation of injected water be maintained?
         │  ├─ NO: (NOT_FEASIBLE) (Excessive mixing expected)
         │  └─ YES: (FEASIBLE_WITH_CONDITIONS) (Requires careful recovery and
         │     buffer management) → Continue to 3.4

3.4 [GEOCHEM_COMPAT] Are there geochemical compatibility concerns?
     (scaling, clogging, contaminant mobilization)
     
     INPUT: (geochem_data), (scaling_studies), (clogging_risk), (mitigation_feas)
     
     DEPENDENCIES:
     
     LOGIC:
     
     OUTPUT:
     ├─ A. Source and aquifer waters compatible
     │  → (PROCEED_TO_FINAL) (Final Decision)
     └─ B. Potential incompatibility
         ├─ Follow-up: Can issues be mitigated by pre-treatment or operational
         │  changes?
         │  ├─ NO: (NOT_FEASIBLE) (Unacceptable Geochemical Risk)
         │  └─ YES: (FEASIBLE_WITH_CONDITIONS) → (PROCEED_TO_FINAL)

----------------------------------------------------------------------
FINAL DECISION
----------------------------------------------------------------------

If all paths lead to YES or Conditionally Feasible:
     → SITE IS HYDROGEOLOGICALLY FEASIBLE
     
If any path leads to NOT FEASIBLE:
     → SITE IS NOT VIABLE
